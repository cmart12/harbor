Copilot Whim

An interactive walkthrough

Welcome. Whim is a Copilot-powered system-tray app: capture Spaces, deploy agents, build and schedule skills. This deck is interactive — try the mockups on each slide.

What is Whim?

  • Spaces — capture ideas, tasks & goals from a global hotkey
  • Agents — Copilot personas that work on your canvas or standalone
  • Skills — reusable instructions you can run & schedule
  • Activity log — your Spaces double as a todo list with history

▶ Use → / Space to advance · ? for help · S for notes

1 · Setup

Install, hotkey, and pick a workspace

Getting set up

  1. Install the Copilot CLI: npm i -g @github/copilot
  2. Launch Whim — it lives in your system tray ⚡
  3. Press Ctrl/Cmd+Shift+Space to summon it
  4. Choose a workspace folder (ideally a git repo)
Prerequisites: Node 20+, a GitHub account with Copilot, and (for cloud agents) gh auth login. The workspace stores Spaces, canvases, the event log, and a SQLite cache.

2 · Creating a Space

Capture by typing or voice — AI refines it

Capture a Space

  • Type a thought and press Enter
  • Or press Space on an empty field to dictate (local Whisper)
  • AI refines it — clean title, client, due date
  • It appears as a card in the Spaces tab
Try it: type into the capture field and press Enter. Watch the raw text get refined into a clean title with extracted client and due date, then slide into the Spaces list.

3 · Built-in agents

Seeded personas for every kind of work

Meet the personas

Click a persona to see what it does and where it runs.

Built-in personas: @agent (general local), @editor (edits canvas via comments), @dev (code via git worktrees), @pr (cloud Copilot coding agent → PRs), @cloud (ephemeral cloud sandbox), @secret-agent (zero-trace), @sandbox (sandboxed policy). Run locations: Local 💻, Cloud ☁️, Copilot Coding Agent 🤖.

4 · Commenting to agents

Select text, @mention, and an agent gets to work

Deploy from the canvas

  1. Open a Space's canvas
  2. Select the text you want worked on
  3. Add a comment and @mention a persona
  4. The agent replies in-thread and edits the canvas
Select text in the fake canvas, type a comment, pick a persona from the @mention dropdown, and submit. The agent shows a running badge, then replies with what it changed.

5 · Standalone agents

Kick off an agent without a canvas

The Workers tab

  • Hit + New Agent — no canvas required
  • Give it a prompt and a persona
  • Watch live steps stream in the Workers tab
  • Approve / deny permission requests inline
Click New Agent, enter a prompt, and watch the worker card stream tool steps, pause for an approval, then complete. Every agent across every Space shows here.

6 · Creating skills

Reusable instructions as SKILL.md

Build a skill

  • A skill is a SKILL.md in .agents/skills/<name>/
  • Give it a name, description, and emoji
  • Create from the Skills tab
  • Run it on demand — or schedule it (next slide)
Fill in name and description; an emoji is auto-assigned. The SKILL.md preview (frontmatter + body) updates live, mirroring how Whim stores skills on disk.

7 · Scheduling skills

Let skills run themselves

Put skills on a schedule

  • Frequency: daily · weekdays · weekly · biweekly · monthly
  • Pick a time (and day of week)
  • See the next run — Whim checks every 60s
  • When due, it runs and spawns a Space
Choose a frequency and time; the "next run" label recomputes live. Trigger it to watch the scheduler create a Space automatically.

8 · Spaces as a todo list

With a logged activity history

Track work & history

  • Spaces flow: captured → in progress → done
  • Toggle complete with a click
  • Every action is appended to an activity log
  • Completed Spaces live in the Past tab
Toggle Spaces between states and watch the activity timeline append entries — the same event log that powers recurrence and the Past view.
